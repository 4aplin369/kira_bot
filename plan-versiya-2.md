# Версия 2 — рабочий чек-лист

Цель этапа: добавить функции, которые **копят данные** (дневник, самочувствие,
напоминания, дата ПДР от подруги, счётчик шевелений с памятью). Сначала всё
тестируем на админе, потом открываем подруге — она просто увидит новые кнопки.

См. общий замысел в [plan-bota-dlya-beremennyh.md](plan-bota-dlya-beremennyh.md),
раздел «Версия 2».

---

## A. Решить до старта (развилки)

- [ ] **Где хранить данные.**
  - Вариант 1 (просто): **SQLite-файл на `/data`** — у Amvera этот путь
    смонтирован как постоянный (`persistenceMount: /data` в `amvera.yml`),
    значит данные переживают передеплой. Для одного маленького бота этого хватит.
  - Вариант 2 (надёжнее): внешний Postgres (**Supabase / Neon**) — переживёт
    даже смену хостинга, есть встроенные бэкапы.
  - 👉 Предлагаю начать с SQLite на `/data`. **Перед этим проверить**, что на
    нашем тарифе Amvera `/data` действительно постоянный (написать тестовый
    файл → передеплой → проверить, что файл на месте).
- [ ] **Фото животика.** Простой путь — хранить в базе только Telegram
  `file_id`, сами фото лежат на серверах Telegram (бесплатно, переотправляются
  по `file_id`). Минус — привязка к токену бота; на всякий случай предусмотреть
  фоновое скачивание-бэкап.
- [ ] **Бэкапы.** Решить, как и куда выгружать копию базы (дамп SQLite раз в
  N дней в облако / в личку админу).

## B. Технический фундамент (один раз, до фич)

- [ ] Подключить слой БД (`aiosqlite` или SQLAlchemy).
- [ ] Спроектировать таблицы:
  - `users` — tg_id, due_date, name, role, created_at
  - `diary` — user_id, week, photo_file_id, note, ts
  - `wellbeing` — user_id, date, mood, symptoms, sleep
  - `reminders` — user_id, type, time, enabled
  - `moves` — user_id, count (или сессии с историей)
- [ ] Инициализация/миграции схемы при старте бота.
- [ ] Многошаговый ввод (`ConversationHandler`) — для даты ПДР и заметок дневника.
- [ ] **Восстановление напоминаний при старте**: бот читает включённые
  напоминания из БД и заново регистрирует их в `JobQueue` (иначе после
  передеплоя они пропадут).

## C. Фичи (по приоритету)

- [ ] **Ввод даты ПДР подругой** + онбординг (сейчас дата зашита вручную в коде).
- [ ] **Счётчик шевелений с сохранением** — вернуть кнопку подруге, починить
  обнуление при передеплое.
- [ ] **Трекер самочувствия** 😊 (настроение / симптомы / сон) + история за неделю.
- [ ] **Дневник** 📖 — фото животика + заметки по неделям + «лента прогресса».
- [ ] **Напоминания** 🔔 (вода, размяться, витамины, врач) — настраиваемые,
  по умолчанию выключены.

## D. Контент (можно параллельно, без кода)

- [ ] «Доброе утро»: расширить до ~60–90 фраз, чтобы не повторялось месяцами.
- [ ] Проверить таблицу недель — все ли недели до 40 на месте.
- [ ] Дополнить «можно/нельзя» и чек-листы при необходимости.

## E. Перед выкаткой подруге

- [ ] Прогнать все новые функции на роли админа.
- [ ] Проверка бэкапа: передеплой не теряет данные.
