# Деплой бота

Бот работает на **polling** — публичный URL/вебхук не нужен. Это фоновый
процесс (worker), который должен крутиться постоянно.

> ⚠️ Запускать можно только **одну** копию бота. Если он работает на хостинге,
> не запускай его параллельно локально — Telegram не даёт двум инстансам
> читать обновления одновременно (ошибка `Conflict: terminated by other getUpdates`).

## Переменные окружения

Файл `.env` в репозиторий не попадает (он в `.gitignore`). На хостинге задай
эти переменные в панели проекта:

| Переменная   | Пример              | Назначение                          |
|--------------|---------------------|-------------------------------------|
| `BOT_TOKEN`  | `123456789:AAE...`  | токен от @BotFather (обязательно)   |
| `DUE_DATE`   | `2026-10-26`        | предполагаемая дата родов (ГГГГ-ММ-ДД) |
| `ADMIN_IDS`  | `122879776`         | Telegram ID тестировщика (через запятую) |
| `FRIEND_IDS` | `78689009`          | Telegram ID подруги (через запятую) |

## Railway

1. New Project → Deploy from GitHub repo (или `railway up` из папки).
2. В разделе **Variables** добавь переменные из таблицы выше.
3. Railway сам поставит зависимости из `requirements.txt` и запустит процесс
   из `Procfile` (`worker: python main.py`).

## Render

1. New → **Background Worker** (не Web Service!).
2. Build command: `pip install -r requirements.txt`
3. Start command: `python main.py`
4. В **Environment** добавь переменные из таблицы.

## Fly.io

Fly разворачивает через Docker. Нужен `Dockerfile` — попроси добавить, если
выберешь Fly (в текущем наборе его нет).

## Проверка после деплоя

В логах хостинга должно появиться:

```
Бот запущен. Текущая неделя: ... Автоутро в 09:00 для {...}.
Application started
```

После этого напиши боту `/start` в Telegram.
